Pressure Monitor

A Krita docker that shows real-time stylus pressure with a circular gauge, scrolling sparkline, peak marker, and live statistics.

Installation

  1. In Krita, open Tools → Scripts → Import Python Plugin…
  2. Select pressure_monitor.zip
  3. Restart Krita
  4. Go to Settings → Configure Krita → Python Plugin Manager and enable Pressure Monitor
  5. Restart Krita again
  6. Open the docker via Settings → Dockers → Pressure Monitor

Interface

ElementDescription
Circular arc Fills clockwise as pressure increases. Color shifts from blue (light) through purple to red (heavy).
Yellow dot Marks the peak pressure reached in the current session.
Sparkline Scrolling line at the bottom of the gauge showing the last 80 pressure samples.
Gradient bar Thin live bar below the gauge for quick at-a-glance feedback.
RAW Instantaneous raw pressure value (0.000 – 1.000).
PEAK Highest pressure recorded since the last reset.
AVG Rolling average of the last 80 samples where the pen was actively pressing. Samples taken while the pen is lifted are excluded.
↺ RESET PEAK Clears the peak marker and resets the AVG history.

Smooth decay

When the pen is lifted, the gauge does not snap to zero immediately. It fades out over ~300 ms to make brief lifts easier to read visually. Only real pen-contact samples affect the RAW and AVG values.

How pressure is captured

Krita's Python API does not expose pen pressure directly. This plugin installs a QTabletEvent filter on the Qt application and reads event.pressure() (range 0.0 – 1.0) from TabletPress, TabletMove, and TabletRelease events. The filter is read-only — it never blocks or modifies events.

Note: Pressure values depend on your tablet driver and the sensitivity curve set in Krita's Settings → Configure Krita → Tablet settings. The plugin reads the post-curve value as Krita sees it.

Troubleshooting

SymptomLikely cause
Pressure always 0% Tablet driver not sending QTabletEvent. Try toggling Use WinTab vs Windows Ink in Krita's tablet settings.
Docker not visible Enable the plugin in Settings → Python Plugin Manager and restart Krita, then check Settings → Dockers.
Krita fails to start after install Check the Krita log for Python import errors. Run Krita from a terminal to see the traceback.
